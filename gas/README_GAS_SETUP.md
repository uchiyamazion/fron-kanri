# GAS統合セットアップガイド

## 概要

既存の「空調点検・作業報告システム」と新規「フロン管理システム」を
**1つのGASプロジェクト・1つのスプレッドシート**に統合します。

## スプレッドシートの構成（統合後）

| シート名 | 用途 | 更新元 |
|----------|------|--------|
| `点検報告` | 作業報告データ（既存・変更なし） | 空調点検システム |
| `機器マスタ` | **両システム共通の機器DB** | 両システム（自動同期） |
| `充填・回収記録` | 充填量・回収量の全履歴 | 自動転記＋手動入力 |
| `法定点検記録` | 法定点検の実施履歴 | フロン管理システム |
| `漏洩量サマリ` | 年度別CO₂換算集計（自動） | GAS自動生成 |
| `fron_data` | フロン管理JSONバックアップ | フロン管理システム |

## セットアップ手順

### Step 1: 既存スプレッドシートを開く
- 空調点検報告システムが使用しているスプレッドシートを開く

### Step 2: Apps Scriptを置き換える
1. 拡張機能 > Apps Script
2. `Code.gs` の内容を **すべて削除**
3. `gas/Code.gs` の内容を **全貼り付け**
4. 保存（Ctrl+S）

### Step 3: 再デプロイ
1. デプロイ > デプロイを管理
2. 既存デプロイの ✏️ 編集 > バージョン「新しいバージョン」
3. デプロイ → URLは変わらない（既存のURLがそのまま使える）

### Step 4: フロン管理システムの.env設定
```env
# 既存のac-inspectionと同じURLでOK
VITE_GAS_URL=https://script.google.com/macros/s/xxxxxxxx/exec
```

## 連携の仕組み

```
空調点検システムで作業報告を保存
    ↓ GAS createRecord / updateRecord
点検報告シートに保存
    ↓ 自動トリガー: autoSyncFillFromInspection()
充填・回収記録シートへ自動転記
    ├── refAdd（追加充填） → kind: 'fill'
    └── refRecover（回収） → kind: 'recovery'
    ↓
フロン管理システムが fron_load を呼ぶと
    └── 点検報告 + 充填・回収 + 機器マスタを統合して返す
```

## 自動転記されるフィールド対応

| 点検報告フィールド | 充填・回収記録 | 説明 |
|-------------------|---------------|------|
| `refAdd` | amount (kind=fill) | 追加充填量 |
| `refRecover` | amount (kind=recovery) | 回収量 |
| `refrigerant` | refrigerant | 冷媒種別 |
| `workDate` | date | 作業日 |
| `worker` | vendor | 担当者→業者名 |
| `customerName` + `systemName` | eqMasterId | 機器マスタに自動紐付け |

## 機器マスタの自動作成ルール

点検報告が新規作成されると、以下の優先順で機器マスタを検索・作成します：

1. `serial`（製造番号）が一致 → 既存機器を更新
2. `customerName` + `systemName` + `model` が一致 → 既存機器を更新
3. 一致なし → **新規機器マスタを自動作成**（ID: EQ-001, EQ-002...）

## 注意事項

- 既存の点検報告データは **そのまま保持** されます
- URLの変更は不要です（既存のデプロイを更新するだけ）
- `eqMasterId` 列が `点検報告` シートに追加されます（新規列）
