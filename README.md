# フロン管理システム — シオンテクノス株式会社

フロン排出抑制法に基づく機器管理・点検記録・漏洩量計算を一元管理するWebアプリです。

## 機能

| 機能 | 内容 |
|------|------|
| 機器台帳 | 設置場所・冷媒種別・充填量・定格能力の管理 |
| 簡易点検 | 3ヶ月ごとのチェックリスト記録 |
| 法定点検 | 1年/3年点検の記録・期限管理（第16条） |
| 充填・回収 | 充填量・回収量・証明書番号の記録 |
| 漏洩量計算 | GWP×充填量でCO₂換算を自動計算（第19条） |
| アラート | 期限超過・7日以内の点検をダッシュボードに表示 |
| CSV出力 | 機器整備記録簿・漏洩量報告書のCSV出力 |

## 技術スタック

- **フロントエンド**: React + Vite
- **データ保存**: Google Apps Script（スプレッドシート） + localStorage（オフライン対応）

## セットアップ

### 1. フロントエンドの起動

```bash
npm install
cp .env.example .env
npm run dev
```

### 2. Google Apps Script の設定

1. [Google スプレッドシート](https://sheets.google.com) を新規作成
2. **拡張機能 > Apps Script** を開く
3. `gas/Code.gs` の内容をエディタに貼り付けて保存（Ctrl+S）
4. **デプロイ > 新しいデプロイ** をクリック
5. 種類: **ウェブアプリ** を選択
6. 設定:
   - 説明: `フロン管理システム`
   - 実行ユーザー: **自分**
   - アクセスできるユーザー: **全員**（社内のみの場合は「自分のみ」）
7. **デプロイ** ボタンをクリック → URLをコピー

### 3. 環境変数の設定

`.env` ファイルを編集:

```env
VITE_GAS_URL=https://script.google.com/macros/s/xxxxxxxx/exec
```

### 4. ビルド & デプロイ

```bash
npm run build
# dist/ フォルダをWebサーバーに配置、またはVercel/Netlifyにデプロイ
```

## データ保存の仕組み

```
ブラウザ操作
    ↓ 即時
localStorage（オフラインでも動作）
    ↓ 2秒後（デバウンス）
Google Apps Script
    ↓
Google スプレッドシート
├── fron_data シート（JSON全体）
├── 機器台帳 シート（人間が読める形式）
├── 充填・回収記録 シート
└── 点検記録 シート
```

- GASが設定されていない場合はlocalStorageのみで動作
- 複数端末での利用時はGAS経由でデータが同期されます

---

## スプレッドシートの構成

| シート名 | 内容 |
|----------|------|
| `fron_data` | JSONデータ（システム用） |
| `機器台帳` | 登録機器一覧（人間が読める形式） |
| `充填・回収記録` | 充填・回収の全履歴 |
| `点検記録` | 簡易点検・法定点検の全履歴 |

## フロン排出抑制法 対応条項

| 条項 | 内容 | 本システムの対応 |
|------|------|-----------------|
| 第15条 | 機器の点検義務 | 機器台帳・簡易点検記録 |
| 第16条 | 法定点検義務（1年/3年） | 法定点検記録・期限アラート |
| 第19条 | 漏洩量の算定・報告 | CO₂換算計算・CSV出力 |
